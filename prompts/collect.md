# 情報収集プロンプト

## 目的

指定されたソロプレナーに関する公開情報から、以下を抽出する：

1. **技術スタックに関する明示的な事実**
2. **ビジネス戦略に関する明示的な事実**
3. **マーケティング手法に関する明示的な事実**

## 重要な制約

- **推測・推論は一切禁止**。情報源に「〜を使っている」「〜で構築」「〜という戦略」などと**明示的に書かれていることだけ**を記録する
- 「〜っぽい」「おそらく」「〜と推測される」といった表現があっても、それは事実ではなく推測として扱う
- 収益額などの具体的数字は、本人が公開している場合のみ記録する

## 入力

以下の情報源テキストを順に処理する：

1. **ブログ記事 / 技術記事**（📄）
2. **GitHub 公開リポジトリ**（🐙）— README、package.json、言語統計など
3. **Podcast / YouTube 文字起こし**（🎙️）
4. **SNS投稿（X等）**（🐦）

## 出力形式

```markdown
# [ソロプレナー名] — 観察メモ

## 📄 ブログ記事
- URL: [リンク]
  - 言及: "[原文の該当箇所を引用]"
  - 抽出事実（技術）: [技術名と用途]
  - 抽出事実（ビジネス戦略）: [戦略名と詳細]
  - 抽出事実（マーケティング）: [手法名と詳細]

## 🐙 GitHub
- リポジトリ: [リポジトリ名]
  - 言及: "[READMEやコードからの該当箇所]"
  - 抽出事実: [使用言語/フレームワーク/ライブラリ]

## 🎙️ Podcast / YouTube
- 出典: [番組名 / 動画タイトル]
  - 言及: "[発言の該当箇所を引用]"
  - 抽出事実（技術）: [技術名と用途]
  - 抽出事実（ビジネス戦略）: [戦略名と詳細]
  - 抽出事実（マーケティング）: [手法名と詳細]

## 🐦 SNS
- 出典: [XポストURL]
  - 言及: "[投稿本文の該当箇所]"
  - 抽出事実（技術）: [技術名と用途]
  - 抽出事実（ビジネス戦略）: [戦略名と詳細]
  - 抽出事実（マーケティング）: [手法名と詳細]

## 未分類・断片的な言及
- [その他、技術名・戦略・マーケティング手法が出てきたが文脈が不明確なものを列挙]
```

## 注意

- 各情報源で同じ技術が言及されていても、**重複して記録する**（後の推測フェーズで複数情報源の有無が確信度判定に使われる）
- 技術のバージョン番号が書かれていれば必ず記録する
- 過去形（「使っていた」）と現在形（「使っている」）を区別して記録する
- ビジネス戦略・マーケティング手法の `name` は簡潔に、補足は `description` に分けることを意識して記録する
