> 作成日: 2026-06-04

## 1. プロジェクト概要

ソロプレナーとして起業したい人向けに、現代の代表的ソロプレナーの**技術スタック・日々の作業・作ったもの**を一覧化・比較できる静的サイト。

### 1.1 サイト名
**Solopreneurs Pages Dev**

### 1.2 コアバリュー
「ソロプレナー間の共通技術スタックを可視化し、閲覧者が自身の学習・採用の指標にできる」

### 1.3 対象ユーザー
- ソロプレナーとして起業したい人
- 技術選定の参考にしたい個人開発者

## 3. 機能要件

### 3.1 一覧画面（カードグリッド形式）

```
┌─────────────────────────────────────┐
│ [国旗] 🇳🇱                           │
│                                     │
│ Pieter Levels                       │
│ オランダ                            │
│                                     │
│ 採用技術:                           │
│ Node.js · React · PostgreSQL · AWS  │
│                                     │
│ [作ったもの] PhotoAI, InteriorAI    │
│                                     │
│ [詳細を見る →]                      │
└─────────────────────────────────────┘
```

- **UI形式**: カードグリッド（レスポンシブ対応）
- **テーブル形式は採用しない**: LLM推測情報を確定事実のように見せないため
- **フィルター機能**:
  - フロントエンド言語で絞り込み
  - バックエンド言語で絞り込み
  - データベースで絞り込み
  - （静的JSによるクライアントサイドフィルタリング）

### 3.2 詳細画面

各ソロプレナーの詳細情報を表示:

1. **基本プロフィール**
   - 名前
   - 国旗（国名のみ）
   - 作ったもの（プロダクト一覧）

2. **技術スタック（中粒度・6属性）**
   | 属性 | 例 | 情報源アイコン |
   |------|-----|--------------|
   | フロントエンド | React / Vue / Svelte | 📄🐙🎙️ |
   | バックエンド | Node.js / Python / Go | 📄🐙🎙️ |
   | データベース | PostgreSQL / MongoDB | 📄🐙🎙️ |
   | ホスティング | AWS / Vercel / Heroku | 📄🐙🎙️ |
   | 主要ライブラリ | Next.js / Tailwind / Prisma | 📄🐙🎙️ |
   | **AI/ML層** | **OpenAI API / Stable Diffusion / Replicate / LangChain / Pinecone** | **📄🐙🎙️** |

3. **開発環境・日々の作業（AI生産性ツール）**
   - コード補完・生成: Cursor / GitHub Copilot / Replit Agent / Windsurf
   - AIアシスタント: Claude / ChatGPT / Gemini / Perplexity
   - UI生成・プロトタイピング: v0 / Bolt / Tempo / Lovable
   - 情報源アイコン付きで掲載
   - ※この情報は本人が明示的に公開していない場合が多く、「非公開」または「不明」の表示になる可能性が高い

4. **X（Twitter）タイムライン埋め込み**
   - 各ソロプレナーの公式Xアカウントの埋め込みタイムライン
   - 肖像権・著作権リスクをXの公式埋め込み機能に委譲

5. **情報源セクション**
   - 推測に使用した公開情報のリンク一覧
   - 各属性の根拠を透明化

### 3.3 免責・注記表示

- サイト全体に **「本サイトに掲載されている技術スタックは、公開情報に基づくLLM推測であり、ハルシネーション（誤情報）を含む可能性があります。確定情報ではありません。」** の注記を常時表示
- 各技術スタック属性に **「推測」ラベル** を付与（色分けまたはアイコン）

## 4. データ収集方針

### 4.1 情報源（公開情報のみ）

| 情報源 | 収集内容 | 記号 |
|--------|---------|------|
| ブログ記事 | 技術スタック言及、開発手法 | 📄 |
| GitHub公開リポジトリ | 言語統計、使用フレームワーク | 🐙 |
| Podcast / YouTube | 技術選定の理由、日々の作業 | 🎙️ |
| SNS（X等） | 技術に関する発言 | 🐦 |

### 4.2 推測方法論

1. 上記情報源から可能な限りテキスト情報を収集
2. 複数情報源をLLMに入力
3. LLMが「最も確率の高い」技術スタックを推測
4. **推測結果には必ず「⚠️ 推測」ラベルを付与**

### 4.3 掲載しない情報

- ❌ **金額情報（MRR、収益、売上など）**: 本人が公開していても掲載しない
- ❌ **居住地の詳細**: 国名のみ掲載。市区町村や地域は非掲載
- ❌ **家族構成・私生活**: 一切掲載しない
- ❌ **プロフィール画像の直接掲載**: X埋め込みのみ使用

### 4.4 誤情報対応

- 本人または第三者からの誤り指摘を受けた場合、**即座に該当情報を削除**
- 訂正・削除依頼の受付窓口（メールまたはX DM）を設置

## 5. プライバシー・法的配慮

### 5.1 肖像権対策

- **プロフィール画像はX（Twitter）タイムライン埋め込みのみ使用**
- Xの公式埋め込み機能を使用することで、肖像権・著作権リスクをプラットフォーム側に委譲
- 画像のダウンロード・自社サーバーでのホスティングは行わない

### 5.2 個人情報保護

- 公開情報のみを収集・掲載
- 金額情報、詳細な居住地、家族構成等は除外
- 国名のみの掲載に限定

### 5.3 著作権・引用

- ブログ記事やPodcastの内容は、**事実情報の抽出に留め、文章の丸写しは行わない**
- 情報源には必ず元URLをリンク

## 6. 技術スタック（本サイト自体）

| レイヤー | 技術 |
|---------|------|
| フレームワーク | Astro v6 |
| ホスティング | Cloudflare Pages |
| スタイル | Tailwind CSS（推奨） |
| フィルター機能 | クライアントサイドJavaScript（Vanilla JS） |
| 外部埋め込み | X（Twitter）Publish ウィジェット |

- **静的サイト生成（SSG）**: MVPとしてビルド時に全コンテンツを生成
- **動的API呼び出しなし**: LLM推測はビルド前のデータ準備フェーズで実行し、出力をJSON/YAMLとして静的ファイルに埋め込む

## 7. 非機能要件

### 7.1 パフォーマンス

- Lighthouseスコア: Performance 90以上
- 初期ロード: 3G回線で3秒以内
- Xタイムライン埋め込みは遅延読み込み（lazy load）

### 7.2 アクセシビリティ

- WCAG 2.1 AA準拠を目指す
- 色分けのみに頼らず、推測情報にはテキストラベルも併記

### 7.3 SEO

- 各ソロプレナーの詳細ページに個別メタディスクリプション
- `noindex` は指定しない（公開情報に基づくため）

## 9. リスク・免責事項

### 9.1 技術的リスク

- **LLMハルシネーション**: 推測情報に誤りを含む可能性が高い
  - 対策: 全推測にラベル付け、情報源透明化
- **情報源の鮮度**: MVPはスナップショットのため、技術スタックの変化に追従しない
  - 対策: スナップショット日を明記

### 9.2 法的リスク

- **誤情報による不利益**: 閲覧者が推測情報を信頼し、技術選定を誤る可能性
  - 対策: 免責事項の明確化、推測であることを強調
- **肖像権・個人情報**: 公開情報であっても、集約・比較という文脈で問題化する可能性
  - 対策: X埋め込みのみ、金額・詳細居住地の除外

### 9.3 運用リスク

- **継続的メンテナンス**: 対象者増加時の工数増大
  - 対策: MVP時点では3名に限定、自動化はPhase 3以降
