# LLM推測用プロンプト

## ファイル構成

| ファイル | 用途 |
|---------|------|
| `collect.md` | 情報収集フェーズ用。公開情報から技術スタックに関する**事実のみ**を抽出 |
| `infer.md` | 推測フェーズ用。観察メモから技術スタックを推測し、JSONを出力 |

## 使用方法

### Step 1: 情報収集

対象ソロプレナーの公開情報（ブログ記事、GitHub、Podcast、SNS）を集め、`collect.md` のプロンプトを使ってLLMに処理させる。

**入力**: 生テキスト（URLや文章をそのまま貼り付け）  
**出力**: 構造化された観察メモ（Markdown）

### Step 2: 技術スタック推測

Step 1 で得た観察メモを `infer.md` のプロンプトとともにLLMに入力し、JSONを出力させる。

**入力**: 観察メモ + 基本情報（名前、国、プロダクト等）  
**出力**: `solopreneurs.json` 形式のJSON

### Step 3: 人間によるレビュー

出力されたJSONを人間が確認し、以下を検証する：

- [ ] 確信度 `low` の属性は推測ラベルが必要なことを認識しているか
- [ ] 情報源（sources）が実際に観察メモに含まれているか
- [ ] 開発ツール（devTools）に推測で値が入っていないか
- [ ] `inferenceNotes` に重要な補足が記載されているか

### Step 4: JSONへの反映

レビュー済みのJSONを `src/data/solopreneurs.json` に反映する。

## 情報源の収集方法（推奨）

| 情報源 | 収集方法 | コツ |
|--------|---------|------|
| 📄 ブログ | 本人のブログ、技術記事、Product Hunt投稿などを検索 | 技術スタックを「どうやって作ったか」語っている記事を優先 |
| 🐙 GitHub | 公開リポジトリの README、package.json、言語統計バー | 組織アカウントより個人アカウントのリポジトリを優先 |
| 🎙️ Podcast | 本人が出演した技術Podcastの文字起こし | "how I built" "tech stack" などのキーワードで検索 |
| 🐦 SNS | X（Twitter）の技術に関する投稿 | 時期が古いものは現在と異なる可能性があるので注意 |

## 注意事項

- **ハルシネーション対策**: `collect.md` で「推測禁止」を徹底し、`infer.md` で確信度を付与することで、誤情報のリスクを最小化する
- **鮮度**: 技術スタックは変化する。観察メモに**情報の日付**を記録し、古い情報（1年以上前）は信頼度を下げて扱う
- **複数プロダクト**: 1人が複数プロダクトを持つ場合、メインプロダクトの技術スタックを優先する。明確に異なるスタックを使っている場合は `inferenceNotes` に記載
