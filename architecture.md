# Architecture

## プロジェクト概要

**Solopreneurs Pages Dev** — 現代の代表的ソロプレナーの技術スタック・開発環境・作ったものを一覧化・比較できる静的サイト。

## データフロー

### ビルド時

1. `src/data/solopreneurs.json` を読み込み
2. Astro が各ページを静的HTMLに生成
   - `index.astro` → `dist/index.html`
   - `[id].astro` → `dist/{id}/index.html`（各ソロプレナー分）
3. Tailwind CSS がユーティリティクラスを最適化
4. 成果物を `dist/` に出力

### ランタイム（ブラウザ）

1. 静的HTMLを読み込み
2. クライアントサイドJSでフィルター機能を実行（Vanilla JS）
3. X（Twitter）埋め込みスクリプトを非同期読み込み

## 主要コンポーネント

### Layout.astro

- サイト全体の共通レイアウト
- ヘッダー（サイト名 + スナップショット日付）
- フッター（免責事項 + 訂正依頼先）
- Xウィジェット用 JS の読み込み

### index.astro

- カードグリッド表示（1〜3カラム、レスポンシブ）
- クライアントサイドフィルター（フロントエンド / バックエンド / データベース）
- 各カード: 国旗、国名、プロダクト、技術スタック、情報源アイコン

### [id].astro

- 基本プロフィール
- 技術スタック6属性（フロントエンド、バックエンド、データベース、ホスティング、主要ライブラリ、AI/ML層）
- 開発環境・AI生産性ツール
- Xタイムライン埋め込み
- 情報源一覧

## 外部依存

| サービス | 用途 | 備考 |
|---------|------|------|
| X（Twitter）| タイムライン埋め込み | 公式Publishウィジェット |
| Cloudflare Pages | ホスティング | 静的ファイル配信 |

## 非機能要件

- **パフォーマンス**: Lighthouse Performance 90以上
- **アクセシビリティ**: WCAG 2.1 AA準拠を目指す
- **SEO**: 各詳細ページに個別メタディスクリプション
- **セキュリティ**: 静的サイトのため特に不要。外部スクリプトは trusted source のみ

## 制約

- **静的出力のみ**: サーバーサイドの動的処理は不可
- **API呼び出しなし**: LLM推測はビルド前に実行し、JSONに埋め込む
- **X埋め込みのみ**: プロフィール画像の自社ホスティングは行わない
